# FleetFlow Architecture

## Overview

FleetFlow is a distributed logistics and delivery orchestration platform.

The system consists of:

* Angular Frontend
* API Gateway
* Order Service
* Delivery Service
* Notification Service
* PostgreSQL
* RabbitMQ
* Redis

---

## High-Level Flow

Customer places an order.

The request goes through the API Gateway and is forwarded to the Order Service.

The Order Service creates the order and publishes an event.

Other services react to the event independently.

This allows services to remain loosely coupled.

---

## Components

### Angular Frontend

Responsibilities:

* Customer dashboard
* Restaurant dashboard
* Rider dashboard
* Admin dashboard

The frontend communicates only with the API Gateway.

---

### API Gateway

Responsibilities:

* Single entry point
* Authentication validation
* Request routing
* Logging
* Rate limiting (future)

The gateway contains no business logic.

---

### Order Service

Responsibilities:

* Restaurants
* Menus
* Orders
* Order lifecycle management

Owns all order-related data.

---

### Delivery Service

Responsibilities:

* Riders
* Delivery assignments
* Delivery tracking

Responsible for assigning riders and tracking delivery progress.

---

### Notification Service

Responsibilities:

* SignalR notifications
* Email notifications
* Event broadcasting

Consumes events generated by other services.

---

### PostgreSQL

Stores:

* Users
* Restaurants
* Orders
* Riders
* Assignments

Primary source of truth.

---

### RabbitMQ

Used for asynchronous communication.

Examples:

* OrderCreated
* OrderAccepted
* RiderAssigned
* OrderDelivered

---

### Redis

Used for:

* Caching
* Fast rider location lookups
* Session-related data (future)

---
